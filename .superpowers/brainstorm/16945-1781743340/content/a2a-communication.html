🔄 A2A Communication Protocol

Bagaimana agent-agent berkomunikasi via MCP

📡 Transport: MCP Native

Setiap agent adalah MCP Server. Orchestrator adalah MCP Client yang manggil agent lain, dan agent juga bisa manggil agent lain.

🖥️ stdio
Default untuk local development. Subprocess MCP server via CLI.
dataforge agent run pipeline --port 9001
🌐 SSE
Untuk production/remote. Agent jalan sebagai HTTP server, komunikasi via SSE.
dataforge agent run dq --host 0.0.0.0 --port 9002

📨 Message Flow

# 1. Orchestrator receives task from user
Orchestrator ← User: "Fix nulls in orders pipeline"

# 2. Intent parsing + task planning
Orchestrator: plan → [pipeline, schema, dq]

# 3. Orchestrator calls Pipeline Agent via MCP tool call
Orchestrator → Pipeline Agent: list_tools()
Orchestrator → Pipeline Agent: execute("scan orders table")
Pipeline Agent → Orchestrator: {result, artifacts, confidence}

# 4. Orchestrator passes context to next agent
Orchestrator → Schema Agent: execute("detect drift on orders", context: {pipeline_result})
Schema Agent → Orchestrator: {drift_report}

# 5. Synthesize results → present to user
Orchestrator → User: "Fixed 3 null columns, detected 2 schema drifts, added 5 DQ rules"

📋 Message Payload Structure

MCP tool call → response cycle membawa konteks + artifacts antar agent:

# Tool call dari Orchestrator ke Agent
{
 "tool": "execute",
 "params": {
  "task": "check data quality on orders",
  "session_id": "sess_abc123",
  "context": {
   "pipeline_result": {...},
   "artifacts": ["schema_drift_report.md"]
  }
 }
}

# Response
{
 "status": "success",
 "summary": "Found 8 null violations in 3 columns",
 "confidence": 0.94,
 "artifacts": {"dq_report.json": {...}},
 "requires_approval": false
}

🔗 A2A Patterns

➡️
Direct Call — Orchestrator → Agent
Orchestrator manggil tool agent langsung. Paling sederhana, cocok untuk sequential pipeline.
agent.call_tool("pipeline", "execute", {"task": "..."})
🌐
Broadcast — Fan-out ke multiple agents
Orchestrator kirim task ke beberapa agent parallel, tunggu semua selesai, merge hasil.
parallel([pipeline.call(...), dq.call(...), schema.call(...)])
🔁
Agent-to-Agent — Direct (via Orchestrator proxy)
Agent A minta Orchestrator untuk forward message ke Agent B. Orchestrator manage context & routing.
orchestrator.forward(from="pipeline", to="dq", payload={...})

🛡️ Human-in-the-Loop

Setiap agent response bisa minta approval sebelum dieksekusi. Orchestrator handle escalation:

# Agent meminta approval
Agent → Orchestrator: {status: "pending_approval", action: "ALTER TABLE orders DROP COLUMN temp"}
Orchestrator → User: ⚠️ "Pipeline Agent wants to drop a column. Approve?"
User → Orchestrator: 👍 yes
Orchestrator → Agent: continue
