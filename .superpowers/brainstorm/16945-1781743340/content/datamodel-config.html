📦 Data Model & Configuration

Section 4: Core types, schemas, and how to configure agents

🏗️ Core Data Model (Pydantic)

# === Core Models ===

class Task:
 id: str
 description: str # "check null rates in orders table"
 context: dict # shared context across agents
 session_id: str
 agent_plan: list[AgentStep] # resolved plan

class AgentStep:
 agent: str # "pipeline-agent"
 tool: str # "execute"
 params: dict
 depends_on: list[str] # step IDs that must complete first
 parallel: bool # can run in parallel?

class AgentResponse:
 status: "success" | "error" | "pending_approval"
 summary: str
 confidence: float # 0-1
 artifacts: dict # output files, reports, etc.
 requires_approval: bool
 error: ErrorInfo | None

class Capability:
 name: str # "data_quality.profiling"
 version: str
 tools: list[str] # tool names this agent provides

⚙️ Configuration

📄 YAML Config
# ~/.dataforge/config.yaml

version: "1.0"
project: "my-data-platform"

agents:
 pipeline:
  command: "python -m d4.agents.pipeline"
  transport: stdio
  capabilities: ["sql", "spark", "dbt"]
 dq:
  command: "python -m d4.agents.dq"
  transport: stdio

integrations:
 warehouse: snowflake
  target: prod
🔧 CLI
# Init project
dataforge init # creates config.yaml

# Start all agents
dataforge start

# List available agents
dataforge agent list

# Install plugin
dataforge plugin install d4-agent-schema

# Run one-off task
dataforge run "profile customers table"
