🛡️ Security, Error Handling & Observability

Section 3: Resilience & safety across all layers

⚠️ Error Handling Strategy

Agent-level Retry
MCP tool call gagal? Agent auto-retry 3× dengan exponential backoff. Kalau masih gagal, laporkan ke Orchestrator.
retry(max=3, backoff=1.5, max_delay=30s)
Graceful Degradation
Kalau Schema Agent down, Pipeline Agent tetap bisa jalan dengan reduced capability. Orchestrator manage fallback routing.
Circuit Breaker
Agent yang repeatedly gagal auto-dimatiin sementara (30s cooldown). Orchestrator cari alternatif atau notify user.

🔒 Security Layers

Tool-level RBAC
Setiap MCP tool punya permission level. read (safe) vs write (butuh approval).
Dry-run Mode
Semua destructive operation bisa dry-run dulu. Output: "akan melakukan X, Y, Z. Approve?"
Audit Trail
Semua tool call, decision, approval tercatat: siapa, apa, kapan, hasilnya.
Credential Isolation
Tiap agent punya credential scope sendiri. Pipeline Agent gak bisa akses DQ Agent's secrets.

📊 Observability

Orchestrator exposes MCP tools for observability:
get_pipeline_status(pipeline_id) → current state, agent progress, errors
get_agent_health() → list all agents, uptime, last_seen, error_count
get_session_logs(session_id) → full audit trail of a session
get_metrics() → tool call count, avg latency, error rate, agent load

Integration dengan OpenTelemetry optional — bisa export traces ke Jaeger, Grafana, dll.

📝 Session Management

# Setiap task lifecycle punya session_id
session = Session.create(task="Fix nulls in orders")
→ session_abc123

# Agent bisa resume context dari session sebelumnya
session.get_context() → {pipeline_result, schema_report, ...}

# User bisa cancel session, rollback changes
session.cancel()
session.rollback() # undo destructive operations where possible
