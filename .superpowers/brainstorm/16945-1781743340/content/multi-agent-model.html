🤝 3 Collaboration Patterns

🔗

Pipeline — Sequential

Output agent A → input agent B → input agent C

Schema Agent: detect schema drift → Pipeline Agent: adjust SQL → DQ Agent: validate output
🌐

Parallel — Fan-out

Satu task, banyak agent independen, results di-merge

Router: "Audit data platform"
├─ Pipeline Agent: scan pipeline health
├─ Schema Agent: check schema consistency
└─ DQ Agent: profile data quality
🔄

Debate — Review & Validate

Satu agent produce, agent lain review/kritik sebelum final

Pipeline Agent: generates SQL fix →
Schema Agent: reviews schema impact →
DQ Agent: validates edge cases →
All: consensus → execute or revise

⚖️ Conflict Resolution

Ketika agent ngasih pendapat berbeda:

  • Weighted voting — tiap agent punya confidence score + authority level
  • Escalation — ke Router Agent untuk arbitrase, atau ke human
  • Auto-retry — kalau result uncertainty tinggi, agent bisa minta re-execute dengan lebih banyak context
