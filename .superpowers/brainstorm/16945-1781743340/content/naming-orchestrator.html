🏷️ Naming & Core Orchestrator

Section 1: Identitas + Jantung Framework

Nama Framework: mcp-dataforge (atau namamu sendiri?)

Nama sementara "DataForge" — Forge = tempa data mentah jadi berkualitas. Tapi kalau ada preferensi lain, ganti aja.

🧠 Orchestrator MCP Server — Detail

Ini adalah entry point utama. Satu MCP Server dengan tools berikut:

# Tools yang di-expose ke client (Claude Code, dll)

@mcp.tool
async def route_task(
 task: str,     # "check null rates in orders table and fix"
 context: dict | None = None
) → TaskResult:
 """Parse intent, plan multi-agent execution, orchestrate & return result"""

@mcp.tool
async def list_agents() → list[AgentInfo]:
 """Discover all available MCP agent servers and their capabilities"""

@mcp.tool
async def get_pipeline_status(pipeline_id: str) → PipelineStatus:
 """Track progress of running multi-agent pipelines"""

🔧 Agent Discovery — Cara Orchestrator Cari Agent Lain

Dua mekanisme discover agent MCP server:

📄 Config-based
agents = {
 "pipeline": {"command": "python -m d4.agents.pipeline"}
 "dq": {"command": "python -m d4.agents.dq"}
}
🔎 Auto-discover

Scan entry_points atau port range. Ideal untuk plugin pihak ketiga. pip install d4-dq-agent → auto detected.

📦 Packaging

pip install mcp-dataforge  # Core orchestrator + built-in agents
pip install d4-agent-schema  # Schema agent plugin (optional)
pip install d4-agent-dq   # DQ agent plugin (optional)
